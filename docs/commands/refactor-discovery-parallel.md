현재 프로젝트의 코드베이스를 서브에이전트를 활용해 영역별로 병렬 분석해줘.
분석 영역 예시: services/, components/, app/, types/, lib/
각 서브에이전트가 담당 영역을 분석하고 요약을 반환해.

docs/ 문서도 읽어줘.

리팩토링이 필요한 대상을 식별해줘:
- 코드 중복
- 과도한 복잡도 (함수 길이, 의존성)
- 일관성 부족 (네이밍, 패턴)
- 성능 개선 포인트
- 타입 안전성 부족

제안 형식:
- 대상 (파일명 또는 모듈)
- 문제점 (1줄)
- 개선 방향 (1줄)
- 난이도 (상/중/하)
- 영향 범위 (수정 시 연쇄 영향 있는 파일)

결과를 docs/REFACTOR-DISCOVERY.md에 저장해.
코드 작성하지 마. 분석과 제안만.
